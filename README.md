# 🚀 가상 면접 사례로 배우는 대규모 시스템 설계 기초 스터디

대규모 시스템 설계 능력을 기르고, 서로의 생각을 공유하며 기술적인 트레이드오프를 심도 있게 토론합니다.

## 🛠️ 깃허브 운영 및 참여 방식 (Fork & PR)

본 스터디는 **Fork & Pull Request** 방식으로 운영됩니다.

1. 본 메인 레포지토리를 본인 계정으로 **Fork**합니다.
2. 로컬에 clone 후, 해당 주차 폴더에 자신의 요약본 또는 발표 자료를 업로드합니다.
    * *파일명 예시:* `chapter-01/zladb.md` 또는 `chapter-01/발표자료.pdf`
3. 작업 완료 후, 메인 레포지토리의 `main` 브랜치로 **Pull Request(PR)**를 보냅니다.
    * *PR 타이틀 예시:* `[X주차] 이름 - 요약 및 발표 자료 제출`
4. 스터디원들의 확인을 거친 후 메인 레포지토리에 반영(Merge)합니다.

## 📜 스터디 규칙

* **정기 모임**: 매주 **목요일 오후 9시** (온라인 및 오프라인)
* **진행 방식**: 
    * 매주 정해진 발표자가 해당 분량의 시스템 설계를 발표합니다.
    * 발표 직후, 해당 주제에 대해 자유롭게 질문하고 토론합니다.
* **자료 제출**: 발표 자료 및 개인 요약본은 **스터디 시작 전날(수요일 자정)**까지 PR 제출을 완료해야 합니다.
