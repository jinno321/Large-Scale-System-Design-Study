안정 해시
Consistent Hashing

서버 추가·제거 시 최소한의 키만 재배치하는 분산 해시 기법. 해시 공간을 원형 링으로 표현해 서버와 키를 동일한 함수로 배치합니다.

01

해시 링 · Hash Ring

해시 공간
0 ~ 2³²−1
원형으로 이어 붙인 정수 공간
서버 위치
hash(IP)
서버명·IP를 해시해 링에 배치
키 담당 서버
시계 방향↻
키에서 시계 방향 첫 번째 서버
SHA-1 등 해시 함수의 출력 공간(0~2³²-1)을 원형 링으로 이어 붙입니다.

서버와 키 모두 동일한 해시 함수로 링 위 좌표를 결정합니다.

키의 위치에서 시계 방향으로 처음 만나는 서버가 해당 키를 담당합니다.

02

서버 조회 · Key Lookup

테이블 행 또는 링 위의 키(주황 사각형)를 클릭하면 시계 방향 탐색 경로와 담당 서버를 확인할 수 있습니다.
03

서버 추가 · 제거 · Scale In/Out

서버 E를 추가할 때, 재배치되는 키는 재배치 E와 이전 서버 사이의 키만입니다. 나머지는 변동 없습니다.
서버 C가 제거되면, 영향 C가 담당하던 키만 다음 서버(D)로 이전됩니다.
04

가상 노드 · Virtual Nodes

하나의 물리 서버를 링 위의 여러 위치에 등록합니다. hash("server-A-vnode-0"), hash("server-A-vnode-1")… 처럼 독립적으로 해시하여 배치합니다. 슬라이더를 올려보세요.
